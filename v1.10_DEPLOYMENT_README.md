## 🆕 v1.10相比v1.9的主要变化

### 部署相关的变化：

1. **项目目录**：
   - v1.9: `/opt/review_analysis_v1.9`
   - v1.10: `/opt/review_analysis_v1.10`

2. **主应用文件**：
   - v1.9: `app_v1.9.py` → systemd使用`app_v1.9:app`
   - v1.10: `app_v1.10.py` → 需要复制为`app.py` → systemd使用`app:app`

3. **新增依赖**：
   - v1.10新增：`pyecharts>=2.0.0`（ECharts图表库）

4. **前端变化**：
   - v1.10使用ECharts.js CDN（替换Plotly）
   - 所有图表都是交互式的

## ⚠️ 重要提示

### 部署前必须完成：

1. **代码准备**：
   - ✅ 确保代码已推送到GitHub（或准备好上传）
   - ✅ GitHub仓库名称建议：`review_analysis_v1.10`

2. **服务器准备**：
   - ✅ 购买阿里云服务器（Ubuntu 22.04）
   - ✅ 配置防火墙（开放80、443、5000端口）
   - ✅ 记录服务器IP、用户名、密码

3. **文件准备**：
   - ✅ 确保`app_v1.10.py`文件存在
   - ✅ 确保`requirements.txt`包含`pyecharts>=2.0.0`

### 部署过程中的关键步骤：

1. **重命名主应用文件**（v1.10特有）：
   ```bash
   cp app_v1.10.py app.py
   ```

2. **systemd服务配置**：
   - 必须使用`app:app`（不是`app_v1.10:app`）
   - 工作目录：`/opt/review_analysis_v1.10`

3. **验证部署**：
   - 检查服务状态：`systemctl status review-analysis`
   - 检查图表是否正常显示（v1.10新特性）
